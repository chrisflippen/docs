A raw inventory file enters the system, gets normalized into a vehicle record, then gets reshaped into a 34-column Vehicle Ads TSV. The diagram traces those three stages; the table below is the per-column lookup.

## Three-stage flow

Every raw row passes through two hops on its way to Google Merchant Center: a column resolver maps and transforms source headers into a normalized vehicle record, then the output transforms reshape that record into the 34-column Vehicle Ads TSV.

## Column map

Every TSV column, where it comes from, and how it is shaped. Markers: ● required, ○ recommended, ◐ free listings only.

| GMC column | From | Transform |
|---|---|---|
| ● id | vin | verbatim 17-char VIN |
| ○ title | year + make + model + trim | build title (Google regenerates and ignores it) |
| ○ description | derived + engine, transmission, drivetrain, interior | build description; sole channel for those four attributes |
| ● link | detail_url | verbatim, else blank |
| ● link_template | detail_url | origin + path + `?store={store_code}` |
| ● image_link | images[0] | parse images, http(s) only, cap 9 |
| ○ additional_image_link ×9 | images[1..] | parse images; at most 8 fill, 9th always blank |
| ● google_product_category | constant | literal `916` (Motor Vehicles) |
| ● brand | make | verbatim; the only standalone structured column for make (title also contains make but is regenerated by Google) |
| ● price | price_cents else msrp_cents + trusted scrape | effective price, downward-only, `%.2f USD`; ≤0 blank |
| ● condition | condition | `new` or `used`; certified folds to `used` |
| ● color | exterior_color | verbatim |
| ● VIN | vin | verbatim; identical to id |
| ● year | year | text |
| ● model | model | verbatim |
| ● mileage | mileage + condition | `<n> miles`; new floors to `1 miles`, used 0 blank |
| ○ trim | trim | verbatim |
| ○ body_style | body_style (raw) | enum map; unmapped blank |
| ○ fuel_type | fuel_type (raw) | enum map; unmatched maps to `other` |
| ○ certified_pre-owned | certified | `true` / `false`; stays false until VDP scrape |
| ○ vehicle_msrp | msrp_cents | `%.2f USD`; ≤0 blank |
| ● vehicle_fulfillment(option) | constant | literal `in_store` |
| ● store_code | dealer store_code (manual GBP) | verbatim, else blank; not from the feed |
| ○ store_address | dealer address fields | concat with spaces |
| ○ vehicle_all_in_price | price/msrp/scrape + fees | all-in price, bidirectional, `%.2f USD`; ≤0 blank |
| ◐ vehicle_option | constant | always blank |

### Behavior notes

- Two layers normalize: the ingest layer writes the raw vehicle record, the output layer reshapes fields to Google's enums at TSV-build time.
- `store_code` and the `{store_code}` link macro come from the dealer's manual GBP code, not the feed; the feed's own store-code column only drives attribution.
- Base `price` is downward-only; `vehicle_all_in_price` is bidirectional and adds dealer fees; non-positive price emits blank, never `0.00`.
- Rows missing VIN, link, image, price, make, model, year, color, or used-vehicle mileage are dropped; duplicate VINs are also pruned.
