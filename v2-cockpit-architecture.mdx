## Key

- The shell is a locked viewport frame — the rail and top bar never move. Only the content region scrolls, and the setup modal overlays only that region, so the chrome stays live during setup.
- Each shell singleton (rail, top bar, search, and the rest) has exactly one sanctioned mount site. An audit fails the gate if one appears anywhere else.
- The design-law gate is the definition of done. Every change runs through it in sequence and must pass before it ships — the audits read the shell tree as their source.
- The cockpit is the presentation layer. No customer data, queue, or credentials live here.
